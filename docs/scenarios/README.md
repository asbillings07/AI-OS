# Scenarios

> Primary question: **What should real usage look and feel like?**

Scenarios are real-world usage stories that act as **living acceptance criteria** for Orion. They are narratives, not wireframes — they capture *intent* and *feeling* so that engineering, design, product, and AI work all align on the same experience. If the built product doesn't hold up when read against these stories, the product is wrong, not the story.

## Anchor scenario

- **[The Mission Control Experience](./mission-control-experience.md)** (#7) — the flagship narrative that defines the core promise, the emotional arc (anxiety → orientation → understanding → trust → focus → confidence → relief), and how recovery works when Orion is wrong. Every other scenario builds on this.

## Planned day-in-the-life scenarios

Each of these should be written as a short narrative and used to pressure-test the design (per #12):

- Office day
- Work-from-home day
- Airport / travel day
- Vacation
- Weekend with family
- Deep-focus morning

Each scenario should show: what Orion surfaces, what it deliberately stays quiet about, and how the experience should *feel* in that context.
